Follow a modular structure to improve maintainability.

Separate:

UI logic

API logic

Business logic

Database logic

Formatting

Use consistent formatting rules:

Consistent indentation

Proper spacing

Clear variable naming

Prefer readable code over overly compact code.

Comments

Comment when code is:

Complex

Non-obvious

Contains business logic rules

Example:

// Prevent duplicate school codes

Avoid commenting obvious code.

Error Handling

Always handle errors gracefully.

Examples:

API request failures

Database errors

Validation errors

Return meaningful error responses.

Testing

Critical functions should include tests when possible.

Examples:

Authentication logic

Data validation

Service functions
